## **1. Installing TypeScript**

There are two common ways to install TypeScript:

| Command                             | Usage                                                                             |
| ----------------------------------- | --------------------------------------------------------------------------------- |
| `npm install -g typescript`         | Installs TypeScript globally so you can use the `tsc` command anywhere            |

To verify installation:

```bash

```

## **2. Basic TypeScript Types**

Common built-in types:

* `number`
* `string`
* `boolean`
* `null`
* `undefined`
* `void`
* `object`
* `Array`
* Tuples (fixed-length arrays with defined types)

### **Variable Declaration**

```ts
let greeting: string = "Hello Mohan";
```

But explicit typing is often unnecessary because **TypeScript can infer types**:

```ts
let greeting = "Mohan";   // inferred as string
let userId = 33333;       // inferred as number
let isLoggedIn = false;   // inferred as boolean
```

## **4. Functions and Type Safety**

### **Without Type Annotation (Not Recommended)**

```ts
function addTwo(num) {
  return num + 2;
}
```

This allows:

```ts
addTwo(2);     // OK
addTwo("2");   // Also allowed -> unintended behavior
```

### **Correct Approach**

```ts
function addTwo(num: number): number {
  return num + 2;
}
```

### Function with `string` Input

```ts
function getUpper(val: string) {
  return val.toUpperCase();
}

getUpper("hitesh");
```

### Function with Multiple Parameters

```ts
function signUpUser(name: string, email: string, isPaid: boolean) {}

signUpUser("Hitesh", "a@gmail.com", true); // âœ… correct
```

## **5. Default Parameters**

```ts
let loginUser = (name: string, email: string, isPaid: boolean = false) => {};

loginUser("h", "h@gmail.com");           // isPaid defaults to false
loginUser("h", "h@gmail.com", true);     // override default
```

## **9. `void` and `never` Return Types**

### **`void`: Function does not return anything**

```ts
function consoleError(errMsg: string): void {
  console.log(errMsg);
}
```

### **`never`: Function never returns (usually throws error)**

```ts
function handleError(errMsg: string): never {
  throw new Error(errMsg);
}
```
